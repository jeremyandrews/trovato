# Story 36.5: ritrovo_cfp Plugin

Status: done

## Story

As a **conference attendee**,
I want to see CFP (Call for Papers) deadline badges on conference listings,
so that I can quickly identify conferences with open submission windows and how much time remains.

## Acceptance Criteria

1. `tap_item_view` renders a CFP badge when a conference has an active `field_cfp_end_date`
2. Badge color indicates urgency: green (>14 days), yellow (7-14 days), red (<7 days)
3. Badge shows remaining days in human-readable format (e.g., "CFP Open -- 20 days left")
4. No badge is rendered when the CFP deadline has passed
5. No badge is rendered for non-conference item types
6. No badge is rendered when `field_cfp_end_date` is missing or zero
7. Plugin only processes conference items (early return for other types)

## Tasks / Subtasks

- [x] Implement tap_item_view to extract field_cfp_end_date from conference items (AC: #1, #5, #6)
- [x] Calculate remaining days from deadline vs. item changed timestamp (AC: #2, #3)
- [x] Render color-coded badge HTML with CSS classes (cfp-badge--open, --closing, --urgent) (AC: #2, #3)
- [x] Return empty string for past deadlines, non-conferences, and missing dates (AC: #4, #5, #6)
- [x] Write unit tests for all badge states (AC: #1-#7)

## Dev Notes

### Architecture

Minimal WASM plugin (164 lines including tests). Single tap: `tap_item_view`. Uses the item's `changed` timestamp as a proxy for current time since WASM plugins cannot access the system clock directly.

Badge thresholds:
- `cfp-badge--open`: >14 days remaining (green)
- `cfp-badge--closing`: 7-14 days remaining (yellow)
- `cfp-badge--urgent`: <7 days remaining (red)
- "CFP Closes Today!": <1 day remaining

Supports both numeric (Unix timestamp) and string-parseable values for `field_cfp_end_date`.

### Testing

6 unit tests covering: non-conference items, missing CFP date, green/yellow/red badge thresholds, past deadline. All tests use `__inner_tap_item_view` (the unwrapped inner function generated by `#[plugin_tap]`).
